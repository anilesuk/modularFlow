🧰 Fixes That Actually Work (and Are Used in Production)
✅ 1. Disable Extra PDF Features Explicitly

Even though you already have tagged: false, also include:

const pdfBuffer = await page.pdf({
  format: "A4",
  margin: { top: "2cm", right: "2cm", bottom: "2cm", left: "2cm" },
  printBackground: true,
  preferCSSPageSize: true,
  displayHeaderFooter: false,

  // Add these:
  tagged: false,
  outline: false,
  annotations: false,
  compress: true,        // Reduces junk stream size
  scale: 1,
});


👉 The key is annotations: false — Chromium sometimes includes /Annot objects that look like hidden scripts.

✅ 2. Clean Metadata (Remove Default Chromium Metadata)

By default, Puppeteer embeds:

Producer: Skia/PDF m... Chromium
Creator: HeadlessChrome


Some scanners flag that as “non-standard generator”.

You can sanitize metadata afterward using a PDF library like pdf-lib or pdfkit:

import { PDFDocument } from 'pdf-lib';

async function sanitizePdf(pdfBuffer) {
  const pdfDoc = await PDFDocument.load(pdfBuffer);
  pdfDoc.setTitle("Curriculum Vitae");
  pdfDoc.setAuthor("Anil Sonavane");
  pdfDoc.setSubject("Professional CV");
  pdfDoc.setProducer("OpenResume Generator");
  pdfDoc.setCreator("CV Generator Service");
  return await pdfDoc.save();
}


Then call it after Puppeteer:

let pdfBuffer = await page.pdf(...);
pdfBuffer = await sanitizePdf(pdfBuffer);

✅ 3. Avoid Data URLs / Embedded Fonts

Check your HTML template — if you have:

<style>
@font-face { src: url(data:font/woff2;base64,...) }
</style>


or

<img src="data:image/png;base64,...">


Replace with file-based or remote URLs, or remove inline fonts/images.
Antivirus heuristics often misinterpret base64 blobs as “hidden executable payloads.”

✅ 4. Use --disable-extensions and Clean Launch Args

Simplify your Chromium launch flags:

const browser = await puppeteer.launch({
  headless: true,
  executablePath: chromiumPath,
  args: [
    "--no-sandbox",
    "--disable-extensions",
    "--disable-gpu",
    "--disable-dev-shm-usage",
    "--disable-accelerated-2d-canvas",
    "--no-zygote",
    "--no-first-run",
    "--hide-scrollbars",
    "--mute-audio"
  ]
});


This ensures Chromium doesn’t attach additional metadata or accessibility layers.

✅ 5. Verify Output Buffer Handling

When saving or sending the PDF:

Use binary-safe methods:

fs.writeFileSync("cv.pdf", pdfBuffer, { encoding: "binary" });


When serving via Express:

res.setHeader("Content-Type", "application/pdf");
res.setHeader("Content-Disposition", "attachment; filename=cv.pdf");
res.end(pdfBuffer);

✅ 6. Optional – Postprocess With Ghostscript

If AV still blocks it, a 100% clean method is to re-render it using Ghostscript, which strips all problematic streams:

gs -sDEVICE=pdfwrite -dCompatibilityLevel=1.4 -dPDFSETTINGS=/printer \
   -dNOPAUSE -dQUIET -dBATCH -sOutputFile=clean.pdf input.pdf


This guarantees no embedded JS, fonts, or annotations remain.

🩺 Diagnostic: How to Confirm the Cause

If you want to confirm why the AV is triggered:

Open the PDF in a text editor (or pdftk / qpdf).

Look for any of these:

/JS, /JavaScript

/StructTreeRoot

/Annots

/EmbeddedFile

/XFA

/Producer (HeadlessChrome)

If you see any of those — that’s what triggered the heuristic flag.

✅ Recommended Safe Final Flow

Here’s a stable, production-grade setup:

const browser = await puppeteer.launch({
  headless: true,
  args: [
    "--no-sandbox",
    "--disable-extensions",
    "--disable-gpu",
    "--disable-dev-shm-usage",
    "--disable-accelerated-2d-canvas",
    "--no-zygote",
  ]
});

const page = await browser.newPage();
await page.setJavaScriptEnabled(false);
await page.setContent(html, { waitUntil: "networkidle0" });
await page.emulateMediaType("print");

let pdfBuffer = await page.pdf({
  format: "A4",
  margin: { top: "2cm", right: "2cm", bottom: "2cm", left: "2cm" },
  printBackground: true,
  preferCSSPageSize: true,
  displayHeaderFooter: false,
  tagged: false,
  annotations: false,
  outline: false,
  compress: true,
});

pdfBuffer = await sanitizePdf(pdfBuffer);

await browser.close();
return pdfBuffer;

🧩 TL;DR — What’s Likely Wrong and How to Fix It
Problem	Fix
Antivirus flags “HeadlessChrome” metadata	Overwrite metadata via pdf-lib
Hidden annotations / tags	Add annotations: false, outline: false, tagged: false
Base64 images or fonts	Remove inline base64 resources
PDF corruption from wrong encoding	Ensure binary write mode or Express buffer sending
Persistent AV false positive	Pass through Ghostscript once
